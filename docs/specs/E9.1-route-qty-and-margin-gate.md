# E9.1 — Route Stop qty + Margin Gate

Feature spec for the E9 follow-on (milestone TBD — Fleet & Routes deepening,
[PRD](../PRD.md)). Terms per [CONTEXT.md](../../CONTEXT.md). Grilled and decided with
the owner on 2026-07-15.
Status: **approved** (owner-approved 2026-07-16 — grill locked, Tech drafted by the Engineer
"Carl"; ADR-0007 + CONTEXT.md landed; issues cut into the E9.1 milestone).

Grill inputs: [#254](https://github.com/…/issues/254) (surfaced from #220 route-editor
table, which shipped the goods × Kup/Sprzedaj/Dostarcz grid without a qty column);
E9 spec [E9-fleet-and-routes.md](E9-fleet-and-routes.md) (the equivalence guarantee this
extends and, for the gate, deliberately excepts).

Scope in one line: route Stop-orders gain an optional per-good **quantity (`qty`, "up to
N")** on buy/sell, plus an optional **Margin Gate (`minMargin`)** on buy orders that makes
a ship wait in port until a good's predicted per-unit margin clears a threshold. Explicit
non-goals: **capital-buffer / thaler-budget caps** (a different unit — separate issue if
ever wanted); **waiting to *sell* for a better price** (deliberately excluded by the owner);
**per-order gate satisfaction** when multiple gated buys share a Stop (v1 is atomic —
extension parked, semantics-compatible); **UI "route-rot" escalation** after a very long
wait (parked → UI issue, see Open items).

## Design

### Core: why qty and why the gate

Today route execution is all-or-nothing (`executeStop`, `tick.ts`): buy greedily fills the
Hold with the affordable share; sell empties the good; deliver moves `min(cargo, need)`.
That serves a single-good loop but breaks the moment a player wants to (A) **split the
Hold** across goods at one port, or (B) **dose a sell** across ports for arbitrage. The
#220 table (rows = goods) visually invites both, yet the second good's row is effectively
dead under greedy fill. `qty` fixes A+B. The **Margin Gate** adds intent the sim cannot
express today: *"don't buy here until carrying this good onward is actually worth it."*

### qty — "up to N"

- Optional `qty` on **buy** and **sell** orders (never deliver):
  - **buy**: fill `min(N, Hold space, affordable)`. `N` is a ceiling; still clipped by Hold
    and Thalers exactly as today, just additionally capped at `N`.
  - **sell**: sell `min(N, on board)`; **carry the remainder onward** (this is goal B —
    dosing across ports).
  - **deliver**: no `qty`; stays `min(cargo, need)`.
- `qty` **absent** ⇒ **today's greedy behavior**, byte-identical. Existing saves and simple
  single-good loops are unchanged.
- **E9 equivalence preserved**: "buy N" / "sell N" are ordinary player Commands, so a route
  with `qty` still issues only Commands a manual player could issue.

### Margin Gate — "wait until it's worth carrying"

- Optional `minMargin` on **buy orders only**. Never on sell (owner: no waiting to sell),
  never on deliver.
- **Predicted per-unit margin** = `sell_price(reference port, good) − buy_price(here, good)`,
  computed with the **same market pricing functions the real Commands use** (no phantom
  margin). The reference price is the **current** price at the reference port — a proxy for
  the future; the prediction may be wrong by the time the ship arrives, and that risk is
  intended.
- **Reference port** = the **next *sell*-stop for that good** along the route, scanning in
  route order from the current Stop and **wrapping the loop** (routes are cyclic, so the
  sell may sit "before" the current Stop in the array but still be next in the voyage).
  deliver is **never** a reference (it yields no Thalers). **No sell-stop for the good on the
  route ⇒ the gate is inactive** (the buy executes normally); the editor warns.
- **While the gate is unmet**: the ship **dwells docked at the port** and re-evaluates each
  tick; it does **not** advance the route index. **Other (non-gated) orders at the same Stop
  execute normally on arrival** — the gate withholds only *stop-advancement*, not siblings.
  When the margin reaches `minMargin`, the buy executes once (up to `N`, clipped by
  Hold/Thalers), the order is satisfied, the ship advances.
- **Indefinite wait is accepted** — the player set the threshold, the player owns the
  consequence. There is **no artificial tick cap**. Two safeguards make this humane, not a
  mystery stall: (1) a **visible UI indicator** — a waiting ship shows *"czeka na marżę ≥ X
  (teraz Y)"* — is a **hard requirement** of this feature; (2) the existing escape hatch — a
  manual `sailTo` auto-suspends the route — remains.
- **Natural counter-pressure (emergent, no code)**: upkeep is a daily flat per-ship charge,
  movement-independent (`chargeUpkeep`, `tick.ts`), so a waiting ship burns upkeep with zero
  revenue. Setting the threshold absurdly high is self-punishing.
- **Multiple gated buys at one Stop ⇒ ATOMIC (v1)**: the ship waits until **all** gates at the
  Stop pass, then fires all gated buys together and advances. Simpler and idempotent; the
  niche cost is that one good's ripe deal can expire while another never matures. Per-order
  capture is a semantics-compatible future extension (does not break saves).

### E9 equivalence: a deliberate exception

`qty` preserves the E9 equivalence guarantee; the **Margin Gate breaks it** — there is no
single manual Command meaning "wait for margin". This is a conscious design exception,
recorded in **ADR-0007** (see Tech), not buried in prose.

### Structural finding that shapes everything

The Margin Gate introduces the **first piece of route state invisible to any function of
`World`**: "docked at my own next Stop, siblings already run, index not advanced." Today that
configuration is unreachable (arrival executes *and* advances in one `runRouteForShip` call,
`tick.ts:172-176`), so a docked assigned ship is never seen mid-tick with `nextStopIndex`
pointing at its own port. The gate creates exactly that state — and it **must be persisted**
or save/load diverges from the no-save timeline (autosave every 24 ticks can land mid-wait;
a reload cannot otherwise tell "just arrived" from "been waiting," re-running non-idempotent
sibling buys). This drives the data-shape and save-format decisions below.

### `src/sim/route.ts` — StopOrder fields + pure resolver

```
StopOrder {
  kind: "buy" | "sell" | "deliver"
  good: GoodId
  qty?: number        // "up to N" ceiling; buy & sell only. Absent ⇒ greedy.
  minMargin?: number  // Margin Gate, buy only. Absent ⇒ no gate.
}
```

Both fields on `StopOrder` (one boring shape), **not** a parallel structure. Validity enforced
at the seam, not the type: extend `isValidRoute` (`commands.ts:74`) to reject `qty` on
deliver, `qty ≤ 0` / non-integer, and `minMargin` on sell/deliver.

Pure resolver (Route template only — no `World`, no market; unit-testable, shared by sim + UI):

```
resolveReferencePort(route, currentStopIndex, good): PortId | null
  for offset in 1..route.stops.length:            // wrap the loop
    idx = (currentStopIndex + offset) % length
    stop = route.stops[idx]
    if stop has an order with kind === "sell" && good === good:
        return stop.portId
  return null                                      // no sell stop ⇒ gate INACTIVE
```

O(stops × ordersPerStop) per gated buy per evaluation — negligible. The current Stop is
skipped structurally (offset starts at 1).

### `src/sim/market.ts` — margin helper (pricing seam)

One pure function next to `quoteBuy`/`quoteSell`:

```
unitMargin(buyEntry, buyBase, sellEntry, sellBase): number
  return quoteSell(sellEntry, sellBase, 1) - <unit ask here>
```

The unit ask uses the **same pricing curve the buy Command uses** (decision: `quoteBuy`,
treating `null` at zero local stock as "gate unevaluable ⇒ keep waiting" — a ship with no
local stock has nothing to buy regardless; see Open items #2). Sole site where margin is
computed; the UI calls the same function (no phantom margin).

### `src/sim/ship.ts` — the stored waiting bit (load-bearing)

```
ShipAssignment {
  routeId, nextStopIndex, suspended,
  waiting?: boolean   // NEW: at nextStopIndex's port, siblings done, gate(s) unmet, index held.
}
```

`waiting` absent/false ≡ today. Genuine hidden state → lives in `World` so save/load is an
identity. Sim stores the correctness bit; the UI **display** stays *derived* (route + live
prices).

### `src/sim/tick.ts` — `executeStop` + `runRouteForShip`

**`executeStop` — qty clipping (byte-identical when qty absent):**
- buy: `maxQty = order.qty === undefined ? holdSpace : Math.min(order.qty, holdSpace)`, then
  `qty = maxAffordableQty(entry, base, maxQty, thalers)` (undefined ⇒ passes `holdSpace`
  exactly as `tick.ts:104` today).
- sell: `have = ship.cargo[good]`; `qty = order.qty === undefined ? have : Math.min(order.qty,
  have)`; dispatch when `qty > 0`. Remainder carried onward.
- deliver: unchanged.

No change to `maxAffordableQty` or the buy/sell Commands (they already take `qty` + `routeId`).

**`runRouteForShip` — the atomic gate hold.** Restructure the "at my Stop's port" branch
(`tick.ts:167-176`) into a small state machine keyed on `asn.waiting`:
- **Fresh arrival** (`!waiting`): a gate passes if `resolveReferencePort` is `null` (inactive)
  **or** `unitMargin ≥ minMargin`.
  - all gated buys pass (or none) → execute gated buys, advance `nextStopIndex` (mod),
    `waiting` stays false (identical to today for un-gated routes).
  - any gated buy unmet → execute **only non-gated siblings**, set `waiting = true`, **do not
    advance**, **do not execute any gated buy**.
- **Waiting poll** (`waiting === true`, still docked): re-evaluate gated buys' gates *only*
  (do not touch siblings). All pass → execute gated buys once, advance, `waiting = false`.
- **Elsewhere / redirect / route-gone**: as today, plus clear `waiting`.

**Why gated buys fire as a deferred atomic group, not in list order**: a forced consequence of
the "siblings on arrival, gated buy waits" semantics — siblings around a gated buy must have
already run, so it cannot execute in list position. Firing all gated buys only on the advance
tick keeps them idempotent (state touched once, on the advance tick) → re-run-safe across the
autosave boundary. `runDockingPhase` already re-polls docked assigned ships each tick; gate
evaluation reads pre-`marketTick` prices (same as the docking-phase Commands) → deterministic.

### `src/ui/HeadquartersPanel.tsx` — editor (#220 table)

`StopRow` (`:151`) gains: a **qty** number input (shown for active buy/sell; blank ⇒ greedy);
a **minMargin** number input (shown only for active buy); an inline **warning** when a buy has
`minMargin` set but `resolveReferencePort(draft, stopIndex, good) === null`. Editor calls the
same `resolveReferencePort` the sim uses. Fleet/route display: a waiting ship shows *"czeka na
marżę ≥ X (teraz Y)"* — Y = live `unitMargin`, derived. `FleetList.tsx` status gains a "czeka"
state.

### `src/store/persistence.ts` — SAVE_VERSION 10 → 11

Both new shapes are additive and absent-safe (absent ⇒ greedy / no-gate / not-waiting), so the
migration is a documented **identity / no-op**:
- `SAVE_VERSION = 11`; `READABLE_VERSIONS = {10, 11}` (one-hop, per precedent `:58`).
- `migrateV10ToV11(rawWorld)`: return unchanged; comment records *why* it is lossless. Bump
  (rather than silently accepting v10 as v11-shaped) to keep "version tracks World shape" honest.

### ADR-0007 — "Routes may wait: the Margin Gate exception to E9 equivalence"

Records: (i) the exception + scope (buy only; qty does *not* break equivalence); (ii)
determinism preserved (pure per-tick gate, no `Math.random`/`Date.now`, ADR-0002/0003 intact);
(iii) the forced ordering consequence (deferred atomic gated-buy group); (iv) the `waiting`
bit as required persistent state.

### Claims → tests (Sagan law 1 — abridged; full table in the grill transcript)

Key load-bearing claims each name a test: qty-undefined byte-identical (baseline deep-equal
run); buy/sell qty clipping; deliver rejects qty; reference resolution wraps / skips deliver /
null-inactive; margin uses the same pricing fns; **gate never advances while unmet**; **siblings
execute once, never re-run while waiting** (the key regression); gate-passes fires once then
advances; **save/load mid-wait is identity**; migration lossless; suspend-while-waiting
round-trip (clearing discipline). Distribution check (E11 telescope): fraction of ship-days in
`waiting` and the gate-never-met tail over 500 seeds × 200 days × N ships.
